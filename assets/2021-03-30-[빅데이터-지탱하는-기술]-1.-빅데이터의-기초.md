## 1-2. 빅데이터의 기술
<img src="https://github.com/lovelyzzkei/lovelyzzkei.github.io/blob/master/assets/img/big_data_2.jpg?raw=true">

### 데이터 파이프라인
데이터를 수집, 처리하는데에는 두 가지 방법이 있습니다. 실시간으로 데이터를 수집하여 처리하는 스트리밍 형과, 데이터를 축적하여 놓고 이를 한번에 정리하여 추출, 처리하는 벌크 형이 있습니다.    
스트리밍 형으로 수집된 데이터 중 실시간으로 처리가 필요한 데이터들은 스트림 처리를 통하여 실시간으로 데이터를 분석하여 그 결과를 시계열 데이터베이스에 저장합니다. 지금 무슨 일이 일어나고 있는지 실시간으로 알 수 있는 것입니다. 한편, 실시간으로 수집은 하였지만 장기적인 데이터의 분석이 필요한 경우에는 배치 처리를 통하여 분산 스토리지에 데이터를 저장합니다. 분산 스토리지는 저장소라고 생각하시면 될 것 같습니다.    
분산 스토리지에 저장한 데이터를 분석해야 되는 일이 생긴다면 분산 데이터 처리를 통해 데이터를 원하는 형태로 또는 필요한 형태로 가공하게 됩니다.    
이렇게 일반적으로 차례대로 전달해나가는 데이터로 구성된 시스템을 데이터 파이프라인이라 부릅니다.    

### 데이터 레이크
파이프라인의 기본형이 바로 데이터 웨어하우스와 데이터 마트를 사용한 것입니다. 수집한 데이터(원시 데이터)를 데이터 웨어하우스에 보관할 수 있는 형태로 가공(ETL)하여 저장하고, 필요한 데이터를 그때그때 추출하여 데이터 마트를 구축하여 데이터 분석 및 작업을 하는 것이 기본적인 프로세스입니다.   
하지만 원시 데이터의 형태가 다양해지면서 이를 데이터 웨어하우스에 저장하는 ETL 프로세스 또한 복잡해졌습니다. 이에 따라 각각의 형태에 대한 가공 프로세스를 만드는 대신 원시 데이터를 가공하지 않고 그대로 저장하는 데이터 레이크가 등장하였습니다. 데이터 레이크에서 필요한 데이터만을 추출하여 데이터 마트를 구축하여 작업을 할 수도 있고, 일회성으로 수작업으로 데이터 분석을 하는 애드 혹 분석 또한 진행할 수 있습니다.
