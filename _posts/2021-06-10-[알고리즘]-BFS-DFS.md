이번에 알아볼 알고리즘은 BFS입니다. 이 알고리즘은 그래프에서 경로를 찾는 문제에서 많이 사용되는 알고리즘입니다. 참고로 그래프는 정점(node)과 간선(edge)으로 이루어진 자료구조입니다. 그래프에 대해서는 추후에 전반적으로 정리를 하겠습니다. 

## 너비 우선 탐색 (BFS, Breadth-First-Search)

너비 우선 탐색은 이름 그대로 그래프의 너비를 우선으로 탐색한다는 것입니다. 깊게 들어가는 것이 우선이 아니라 **넓게 보는 것**이 우선이 되는 탐색기법으로, 현재 깊이에 있는 노드들을 모두 확인한 뒤 다음 깊이로 이동하여 이를 반복하는 탐색기법입니다. 그림으로 표현하면 다음과 같습니다.

그림에서 볼 수 있듯이 BFS는 시작 노드(루트 노드)에서 가까운 노드들을 우선으로 탐색합니다. 따라서 보통 **두 지점 사이의 최단 경로**를 구할 때 많이 사용되는 알고리즘입니다. 

### 파이썬을 이용한 구현

BFS는 **큐(Queue)**를 이용하여 구현을 하는 것이 일반적입니다. 원리는 다음과 같습니다.  

1. 큐에 시작 노드를 삽입
2. 큐에 있는 노드를 하나씩 빼면서 해당 노드 주변에 있는 노드를 탐색
3. 본인이 원하는 노드가 있다면 그 노드들을 큐에 다시 삽입
4. 1~3의 과정을 원하는 노드가 나올 때까지 반복

의사 코드(pseudocode)로 표현하면 다음과 같습니다.

```python
def bfs(x):
    dq = deque([x]) # 1. 큐에 시작 노드 삽입
    visited = []    # 해당 노드를 방문했는지 여부를 저장하는 배열
    height = 0      # 현재 그래프의 높이

    while dq:   
        qlen = len(dq) # 현재 높이에 있는 모든 노드들을 탐색 (BFS의 정의)
        while qlen:
            cur = dq.popleft()  # 2. 큐에 있는 노드를 하나씩 빼면서

            if cur == 내가 원하는 노드:     # 4. 내가 원하는 노드가 등장하였을 경우 탈출
                return

            # 2~3. 주변의 노드를 탐색하면서 내가 원하는 조건의 노드를 다시 큐에 삽입
            for node in cur_주변:   
                if 문제의 조건 or 내가 원하는 노드의 조건:
                    ...
                    dq.append(노드)
            
            qlen -= 1

        height += 1
```

보통 우리가 탐색을 할 때 이전의 노드들, 즉 한번 방문했던 노드들은 다시 방문하지 않기를 원합니다. 그래야 함수가 무한 루프에 걸리지 않고, 그래프에서 **계속해서 밑으로 내려갈 수 있기 때문입니다.** 그래서 BFS에서는 **노드의 방문 여부**를 처리하는 것이 중요합니다.   

BFS를 공부하며 알아두면 좋은 몇 가지 팁들을 알려드리자면...  

+ **왠만해서는 덱(deque)를 쓰자.**    
    + 참고 링크: [파이썬 collections deque 함수 (덱,데크) 사용법](https://j-ungry.tistory.com/189)   

이거는 파이썬에 해당하는 부분입니다. 파이썬에서 리스트로 큐를 구현할 수 있지만 리스트는 행렬로 구현이 되어 있기 때문에 내장 함수인 `append()`와 `pop`의 시간 복잡도가 $O(N)$에 해당합니다. 큐의 두 가지 연산, `push`와 `pop`이 이루어질 때, 특히 `pop`의 과정에서 앞의 것을 지우고 뒤에 있는 요소들을 모두 한칸씩 앞으로 당겨야하기 때문에 이에 대한 시간이 $O(N)$만큼 소요됩니다.  

하지만 파이썬에서 지원하는 모듈 중 `collections` 내부에 `deque` 라이브러리가 구현되어있습니다. `deque`은 **양방향 큐**로 앞, 뒤에서 `push`와 `pop`이 가능한 자료구조입니다. 이 자료구조는 배열이  아닌 **연결 리스트(Linked-List)**로 이루어져 있기 때문에 위와 같은 번거로운 작업이 필요없어져 `push`와 `pop`에 **$O(1)$**의 시간만 소요됩니다. 

프로그램의 속도가 중요한 코딩 테스트에서 많은 부분에 있어 덱(deque)이 일반 리스트보다 많은 강점을 가집니다. 하지만 덱이 리스트보다는 많은 메모리를 요구한다는 것은 알고 계시기 바랍니다.  

+ **방문 처리가 중요하다.**

여러 BFS 문제들을 풀면서 느낀 점은 BFS를 구현함에 있어서 내가 방문했던 노드들에 대한 **방문 여부를 처리**하는 것이 굉장히 중요하다는 것입니다. 까먹고 이를 구현하지 않거나, 문제의 조건에 맞지 않게 어설프게 구현한다면 문제가 산으로 가게 됩니다.   

문제를 풀다가 또 팁들이 생각나면 적도록 하겠습니다. 

BFS의 시간 복잡도는   
> N은 노드, E는 간선일 때, 인접 리스트로 구현했을 경우 $O(N+E)$
