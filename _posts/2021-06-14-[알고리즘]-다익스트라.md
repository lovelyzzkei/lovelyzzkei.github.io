이번 시간에는 그래프에서 정점 간의 **최단 경로**를 구할 때 대중적으로 많이 사용되는 **다익스트라(Dijkstra) 알고리즘**에 대해서 공부해보겠습니다.  

## 다익스트라 알고리즘 (Dijkstra)

다익스트라 알고리즘은 네덜란드 컴퓨터과학자 에츠허르 다익스트라(Edsger W. Dijstra, 1930~2002)가 고안한 알고리즘으로 **음의 가중치가 없는 그래프의 한 정점(Vertex)에서 모든 정점까지의 최단 거리를 구하는 알고리즘**입니다.   

우리가 앞에서 배웠던 BFS도 그래프에서 최단 경로를 탐색하는 알고리즘이었습니다. 그러면 BFS와 다익스트라의 다른 점은 무엇일까요? 바로 BFS는 **그래프의 간선의 가중치를 무시**한 최단 경로입니다. 좀 더 정확하게 이야기하자면 그래프 내 간선의 가중치가 모두 **동일**하다고 이야기 할 수 있겠습니다. 그렇기 때문에 우리가 앞에서 BFS를 공부할 때, 그래프의 간선은 생각하지 않고 정점의 방문 여부를 중요시하여 최단 경로를 구하였습니다. 하지만 이제는 간선의 가중치도 생각을 해주어야합니다. 그럼 어떻게 구할 수 있을까요?  

다익스트라 알고리즘은 기본적으로 **그리디(greedy) 알고리즘**에 원리를 따릅니다. 그리디 알고리즘에 대하여 간략하게 설명하자면 다음과 같습니다.  

> *매 선택에서 **지금 이 순간 당장 최적인 답을 선택**하여 적합한 결과를 도출하는 알고리즘*

다익스트라 알고리즘은 최단 거리를 구하는 알고리즘이라고 하였습니다. 그러면 어떤 두 정점 사이의 최단 거리를 구하는데 있어서 각 순간의 최적인 답은 무엇일까요? 제가 내린 결론은 이렇습니다.  

> 각 순간은 두 정점 사이의 최단 거리를 구성하는 정점들이고, 이때 최적의 답은 정점들 사이의 거리가 최소인 경우이다.  

쉽게 이야기하자면 두 정점의 최단 거리는 그 최단 거리의 경로에 포함되어 있는 정점들의 최단 거리로 구성되어 있다는 것입니다(최단 거리는 최단 거리들로 이루어져있다!). 그러면 이 알고리즘을 한번 구현해봅시다.  

### 파이썬을 이용한 구현

먼저 시작 노드에서 다른 모든 노드까지의 거리를 저장할 배열을 하나 준비해줍시다. 그리고 우리가 원하는 것은 **최단 거리**이므로 초기값들은 적당하게 큰 값들로 잡아줍시다. 그리고 시작점에서 시작점까지의 거리는 0이므로 `d[start] = 0`을 해줍시다.

```python
def dijkstra(start):
    INF = int(1e9)      # 1000000000
    d = [INF] * (V+1)   # V는 정점의 개수. 정점 번호 1~V라 가정
    d[start] = 0
```

이제 시작점에서 갈 수 있는 노드들을 가져와 거리를 비교해줍시다. 지금 노드까지의 거리에 지금 노드까지의 거리에서 다음 노드까지 가는 거리를 합한 것(d[지금] + 다음-지금 거리)이 현재 저장되어있는 다음 노드까지의 거리(d[다음])보다 작다면 전자의 거리를 갱신해주어야 할 것입니다. 현재 노드를 u, 다음 노드를 v, 그 거리를 w라 할때, 위의 식을 다음과 같이 나타낼 수 있습니다.  

> *if d[u] + w < d[v]: d[v] = d[u] + w*

여기서 위에서 얘기했던 다익스트라 알고리즘의 특성을 생각해봅시다. 다익스트라 알고리즘은 **최단 거리들로 구성되어있는 최단 거리**라고 얘기했었습니다. 그러면 우리가 지금 노드에서 다음 노드들을 생각할 때, **거리가 가장 짧은 노드**를 받는 것이 위의 특성에 알맞을 것입니다. 그리고 이것이 다익스트라 알고리즘의 시간 복잡도를 줄여주는 중요한 아이디어입니다. 이를 위하여 우리는 **우선순위 큐(Priority Queue)**를 사용합니다. 우선순위 큐는 넣을 때는 마음대로인 큐이지만, 뺄때는 **우선순위 순서대로** 빼는 자료구조입니다. 일반적으로 크기가 작을수록 우선순위가 높습니다.  
따라서 우선순위 큐에 (거리, 노드) 쌍들을 넣으면 알아서 거리가 가장 짧은 노드가 제일 앞에 있게 되므로 우리가 원하는 노드를 쉽게 뺄 수 있습니다. 파이썬에서는 `heapq` 모듈을 사용합니다. 코드로 작성하면 다음과 같습니다. 

```python
    q = []
    heapq.heappush(q, (0, start))

    while q:
        dist, now = heapq.heappop(q)      # 현재 노드 반환
        if d[now] < dist:   # 현재 저장되어 있는 거리가 더 짧은 상황이면 갱신 x
            continue

        for v, w in graph[now]:     # graph: {1:[[2, 3], [4, 5]], ...} 형태의 딕셔너리로 가정
            cost = dist + w
            if cost < d[v]:
                d[v] = cost
                heapq.heappush(q, (cost, v))
```

아래는 간단한 그래프에 다익스트라 알고리즘을 적용하는 것을 진행 순서대로 나타낸 것입니다.

당분간 최단 거리 알고리즘들을 올릴 것 같습니다. 다익스트라는 굉장히 유용하지만 **음의 가중치가 없는 그래프**에서만 잘 작동한다는 한계점이 존재합니다. 그 이유를 간단히 얘기해보면 위와 같이 사이클(cycle)이 존재한 그래프에서 음의 간선이 있다고 생각해봅시다. 양의 간선들만 존재할 경우 사이클을 돌면 돌수록 거리가 늘어만 가기 때문에 최단 거리를 구하는데에 있어 아무 영향을 미치지 못합니다. 하지만 음의 간선이 존재할 경우 돌면 돌수록, **더하면 더할수록 값이 계속해서 작아지기 때문에** 무한 루프에 빠지게 됩니다. 음의 간선이 존재할 경우에는 **벨만-포드 알고리즘**을 사용하게 됩니다. 다음 시간에 알아보도록 하겠습니다. 감사합니다.
