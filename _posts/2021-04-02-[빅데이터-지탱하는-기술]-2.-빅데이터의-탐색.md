## 2-1. 크로스 집계
<img src="https://github.com/lovelyzzkei/lovelyzzkei.github.io/blob/master/assets/img/big_data_3.jpg?raw=true">

위 그림에서 왼쪽에 있는 표처럼 행과 열이 있고, 둘의 교차지점에 값들이 존재하는 표를 **크로스 테이블**이라고 합니다. 크로스 테이블은 보기에는 편하지만 데이터베이스에서 처리하기에는 좋지 않은 형식의 테이블입니다. 데이터베이스에 행을 추가하기는 쉽지만 열을 추가하기는 쉽지가 않기 때문입니다.   
따라서 우리는 데이터를 수집할 때 왼쪽과 같은 테이블이 아닌 오른쪽과 같이 열의 항목들이 고정되어있고 행으로만 데이터가 추가되는 형식의 테이블을 사용해야 합니다. 오른쪽과 같은 테이블을 우리는 **트랜잭션 테이블**이라고 부릅니다.  
하지만 우리는 데이터를 수집하는데에 목적이 있지 않습니다. 그 데이터들을 처리하여 분석하고 시각화함에 목적이 있습니다. 따라서 트랜잭션 테이블을 크로스 테이블로 전환하는 작업이 필요합니다. 그 작업을 **크로스 집계**라고 합니다.

### 피벗 테이블
소량의 데이터를 크로스 집계하는데에는 엑셀의 피벗 테이블 기능을 사용합니다. 파이썬 또한 엑셀의 기능들을 사용할 수 있으므로 파이썬의 Pandas 라이브러리를 사용하여 피벗 테이블을 만들어 보도록 하겠습니다. 사용할 엑셀 파일은 아래에 첨부하겠습니다.  

<a href="https://github.com/lovelyzzkei/lovelyzzkei.github.io/blob/master/assets/판매 데이터.xlsx" download>판매 데이터</a>  

```python
import pandas as pd

# 두 시트를 상품ID 칼럼에 대해 합침
df1 = pd.read_excel('판매 데이터.xlsx', '판매 이력')
df2 = pd.read_excel('판매 데이터.xlsx', '상품정보')

df3 = pd.merge(df1, df2, on='상품ID')

# 트랜잭션 테이블을 피벗 테이블로 변환
df4 = pd.pivot_table(df3,                          # 피벗 테이블로 전환할 data frame 
                    index=['상품명', '점포ID'],     # 행 위치에 들어갈 항목
                    columns='매출일',               # 열 위치에 들어갈 항목
                    values='금액')                  # 행과 열이 교차하는 지점에 들어갈 데이터

print(df4)
```

### SQL에 의한 테이블의 집계
Pandas를 이용하여 피벗 테이블로 크로스 집계를 하면 적은 양의 데이터에 대해서는 처리가 쉽지만, 데이터의 양이 많아지면 속도의 저하로 인하여 사용하기가 힘듭니다. 따라서 대량의 데이터에 대해서는 **SQL의 집계함수**를 사용하여 데이터의 양을 줄인 후 그 데이터들을 크로스 집계하여 임의의 크로스 테이블을 얻을 수 있습니다.    
이렇게 SQL을 가지고 먼저 데이터를 집계하고(데이터 집계의 프로세스), 이 데이터를 시각화도구로 크로스 집계(시각화 프로세스)하는 것이 시스템을 구성하는 기본입니다.
