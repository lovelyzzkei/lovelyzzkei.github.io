## 선형 회귀(Linear Regression)

우리는 세상에서 어떤 하나의 요인이 다른 요인에 영향을 끼치는 상황들을 많이 볼 수 있습니다. 공부하는 시간이 많을수록 (대체로) 시험 결과가 좋다든지, 운동하는 양이 많을수록 근육량이 더 많이 증가한다든지 등 우리는 이 세상의 많은 요인들의 영향 속에서 살아가고 있습니다. **선형 회귀(Linear Regression)**는 이 요인들의 관계를 **선형(linear)의 관계**로 모델링하는 것을 말합니다. 조금 더 일반화 시켜서 얘기해보면 

> *독립 변수 $x$와 이에 따라 변하는 종속 변수 $y$의 선형 관계를 모델링하는 것*

이라고 얘기할 수 있습니다. 이때 이 선형 관계에 기본 식이 그 유명한

> $y=Wx+b$

입니다. 독립 변수가 늘어날 수록 $W$의 개수도 증가하겠지요. 그러면 선형 회귀의 과정을 살펴봅시다.  

<img src="https://github.com/lovelyzzkei/lovelyzzkei.github.io/blob/master/assets/img/linear_reg.jpg?raw=true">

### 가설(Hypothesis)

데이터가 주어지고 이들의 선형 관계를 모델링하고 싶다고 컴퓨터에게 얘기하면 컴퓨터는 데이터 간의 관계를 나타낼 수 있는 선형 관계의 식을 유추하기 시작합니다. 이때 이 선형 관계의 식을 **가설(Hypothesis)**라고 합니다. 선형 회귀의 가설은 위에서도 얘기했지만

> $H(x)=Wx+b$

입니다. 그러면 이제 가설을 세웠으니 해야할 일은 데이터에 적합한 $W$와 $b$를 찾는 것입니다.  

### 비용 함수(Cost function) / 손실 함수(loss function)

그러면 어떻게 적합한 $W$와 $b$를 찾을 수 있을까요? 생각해봅시다. 위에서 가설로 세운 식이 한번에 데이터들을 잘 표현할 수 있지는 못할 것입니다. 그러면 가설이 데이터들을 잘 표현한다는 것은 무슨 의미일까요? 가설이 예측한 값과 실제값이 유사할수록 데이터를 잘 표현한다고 얘기할 수 있지 않을까요? 그러면 이는 **예측값과 실제값의 차이**가 작을수록 데이터를 잘 표현한다고 말할 수 있을 겁니다. 즉, 오차의 크기가 작을수록 모델의 정확도가 커질 것입니다.  
이때, 오차를 표현하는 식을 **비용 함수(Cost function)** 또는 **손실 함수(loss function)** 또는 **목적 함수(Objective function)**라고 부릅니다. 세 함수 모두 오차를 표현하는 함수를 의미하기에 혼용해서 사용하겠습니다.   

> $cost(W, b)=\frac{1}{n}\sum_{i=1}^{n}[y^i-H(x^i)]^2$

그리고 회귀 문제에서는 보통 비용 함수를 **평균 제곱 오차(Mean Squared Error, MSE)**를 사용합니다. 왜 이 함수를 사용할까요? 오차를 표현하는 가장 직관적인 방법은 오차의 크기를 구하는 것입니다. 하지만 오차에는 음수도 있기 때문에 이를 양수로 바꾸기 위하여 **제곱**을 해줍니다. 제곱을 해주는 또 하나의 이유가 있는데 바로 터무니없게 예측한 값들은 오차를 더 크게 만들어 그에 대한 페널티를 제공하기 위함입니다. 그리고 이를 **평균**낸 값을 오차로 많이 사용합니다.  
우리는 오차를 구했습니다. 그러면 이제 이 오차를 최소화시켜야겠죠?

### 옵티마이저(Optimizer): 경사하강법(Gradient Descent)

우리의 목적은 최적의 $W$와 $b$를 찾는 것입니다. 설명의 편의를 위하여 최적의 $W$만 찾는다고 하고 비용함수를 그래프로 그리면 위 그림과 같이 이차함수의 형태가 나옵니다. 이때 비용의 최솟값은 **함수의 꼭짓점**에 존재합니다. 그리고 이는 다시 말하면 **비용 함수의 기울기가 0**인 곳이 비용이 최소가 되는 지점이며 우리가 원하는 $W$가 있는 지점입니다. 이것이 바로 오차를 최소화하는 알고리즘인 **경사하강법(Gradient Descent)**의 핵심입니다.  
경사하강법은 구한 cost의 지점에서 접선의 기울기가 작아지는 방향으로 $W$를 수정해가며 최적의 $W$를 얻는 알고리즘입니다.  

> $W := W - \alpha \frac{\partial }{\partial W}cost(W)$

여기서 알파는 앞에서 얘기한 하이퍼파라미터인 **학습률(learning rate)**를 의미합니다. 학습률이 너무 크면 $W$가 발산해버리고, 너무 작으면 오히려 원하는 지점에 다다르지 못합니다. 그렇기에 적당한 학습률을 주는 것이 좋습니다. 보통 0.01 또는 0.001 정도를 줍니다.   
선형 회귀의 과정은 이렇게 끝입니다. 남은 건 모델이 오차를 최소화하는 과정을 계속 반복하면서 우리가 원하는 `접선의 기울기가 0`이되는 지점의 $W$를 구해내는 것입니다.

## 실습

선형 회귀에 대해 알아보았으니까 간단한 예제를 가지고 이를 구현해보겠습니다. 공부하는 시간과 그 시간에 따른 성적 간의 관계를 머신 러닝에서 자주 사용되는 `Keras` 라이브러리를 가지고 구현해보겠습니다.  

```python
# 데이터 준비
X = [1,2,3,4,5,6,7,8,9] # 공부하는 시간
y = [11,22,33,44,53,66,77,87,95] # 각 공부하는 시간에 맵핑되는 성적
```

먼저 데이터를 준비해주겠습니다. 각각의 X와 y는 가장 간단한 1차원 벡터의 형태를 띄고 있습니다. 

```python
# 필요한 모듈들 import
import matplotlib.pyplot as plt
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Dense
from tensorflow.keras import optimizers


# 1. 모델 생성
model = Sequential()

# 2. 모델에 필요한 정보 추가
model.add(Dense(units=1,                # 입력값인 X의 차원
                input_dim=1,            # 출력값인 y의 차원
                activation='linear'))   # 사용할 함수(가설). 선형회귀 -> linear

# 3. 오차 최적화 알고리즘 설정
sgd = optimizers.SGD(lr=0.01)

# 4. 모델에 비용함수를 엮어줌 (compile)
model.compile(optimizer=sgd,           # 최적화 알고리즘
              loss='mse',               # 비용함수
              metrics=['mse'])          # 모델을 평가할 값, 척도, 평가지표
                                        # 앞에서 얘기한 정밀도, 재현율 등이 여기에 속함

# 5. 학습 진행.
model.fit(X, y, 
          batch_size=1,         
          epochs=300,           # 오차 최소화 작업(훈련 횟수)을 300번 수행
          shuffle=False)

# 오차를 최소화하는 직선 시각화
plt.plot(X, model.predict(X), 'b', X, y, 'k.')
plt.show()

# predict(): 학습이 완료된 모델이 입력된 데이터에 대해서 어떤 값을 예측하는지 보여줌
print(model.predict([9.5]))
```

먼저 해주어야 할 것은 학습시킬 모델을 생성하는 것입니다. `Keras`에서는 `Sequential()`이 그 역할을 해줍니다.   
모델이 생성되었으면 그 모델에 필요한 정보들을 주어야겠지요. 케라스에서는 `add()` 함수를 이용하여 모델을 구성하는 **레이어**를 설정해줍니다. `units`, `input_dim`, `activation` 파라미터에 각각 입력값과 출력값의 차원과 사용할 가설을 알려줍니다.  
우리의 목표는 데이터에 **적합한 $W$와 $b$를 찾는 것**입니다. 이를 위해 비용함수와 오차 최적화 알고리즘을 모델에 세팅해주어야 합니다. 위에서 배운 **평균 제곱 오차**를 비용함수에 해당하는 `loss`에, **경사하강법**을 최적화 알고리즘에 해당하는 `optimizer`에 넘겨주어 모델의 학습과정을 설정해줍니다. 케라스에서는 `compile()`이라는 함수를 사용하여 이 작업을 수행합니다.  
가설도 설정하였고 이에 맞는 비용함수와 최적화 알고리즘도 설정하였다면 이제 모델을 학습시키는 일만 남았습니다. `epochs` 파라미터에 얼마나 훈련을 할 것인지를 설정하여 케라스의 `fit()` 함수로 모델 학습을 진행할 수 있습니다. `fit()` 함수의 또 하나의 파라미터인 `batch_size`는 뒤에서 더 자세하게 살펴보겠습니다.
