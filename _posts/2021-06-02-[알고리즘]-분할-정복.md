### 알고리즘

위 블로그에 있는 글들을 읽으며 알고리즘과 코딩 문제를 대하는 저의 자세를 바꿀 수 있었습니다. 자세한 내용은 위의 블로그에 있는 알고리즘 포스팅들을 읽어보시는 것을 추천드리겠습니다. 간단하게 요약하면, 새로운 문제를 만났을 때 항상 **Design(설계) & Analysis(분석)**을 해야된다는 것입니다. 이는 코딩 문제뿐만 아니라 실생활의 여러 문제들에서도 적용할 수 있는 기법입니다.  

+ Design(설계): 이 문제를 풀기 위한 **효율적인** 알고리즘은 무엇들이 있는가?
+ Analysis(분석): 위의 알고리즘들 중 어떤 것이 가장 나은가?

일반적으로 쉬운 코딩 문제들을 풀때는 '아 이렇게 풀면 되겠네' 라고 위의 두 단계가 빠르게 머릿 속에서 지나가며 답이 나오지만, 문제의 난이도가 올라갈수록 어떻게 이 문제를 풀어야할까라는 막막함이 먼저 듭니다. 그럴때 위의 두 단계를 적용시키며 차근차근 풀면 해답이 보일 것이라 생각합니다. 그래서 위의 두 단계, **Design & Analysis**를 체화시키는 것이 중요합니다. 앞으로 제가 올리는 백준 문제들의 풀이에도 최대한 이 두 단계를 적용시켜 풀어보도록 하겠습니다.  

그리고 위의 두 질문은 다음의 하나의 질문으로 합칠 수 있습니다.  

> *어떤 자료구조를 활용해야 문제를 효율적으로 풀 수 있을까?*

그래서 알고리즘과 자료구조는 뗄레야 뗄 수 없는 관계입니다. 파이썬에도 리스트, 큐, 덱 등 다양한 자료구조들이 내장 라이브러리에 구현되어 있지만 각각의 장단점이 존재하고, 주어지는 문제에 알맞는 자료구조들이 존재합니다. 따라서 효율적인 알고리즘을 짜기 위해서는 자료구조에 대한 지식이 풍부해야합니다. 저도 자료구조 강의를 수강한지 거진 2년이 다되서 많이 까먹었지만 알고리즘과 백준 문제들 포스팅을 올리면서 해당 포스팅에 사용한 자료구조들에 대한 설명도 같이 해보겠습니다. 

서두가 길었습니다. 분할정복에 대하여 알아보죠.

### 분할 정복(Divide and Conquer)

분할 정복은 말 그대로 **문제를 분할하여 각각의 문제를 풀고, 그 문제들의 결과값을 합쳐 최종적인 답을 도출**하는 알고리즘입니다. 그래서 많은 책들과 글들에서 다음의 세 과정을 얘기합니다.  

1. Divide - '2개 이상'의 작은 문제(subproblem)로 나눔
2. Conquer(Solve) - 작은 문제들을 품
3. Combine - 답을 합침

여기서 중요한 것이 **Conquer != Divide** 라는 것입니다. 문제를 작은 문제들로 나누었으면, 그 **작은 문제들에 대한 알고리즘을 다시 세워서** 답을 얻고, 그 답을 합치는 것이 분할 정복의 기본적인 개념입니다. 하지만 코딩 문제들에서는 입력으로 들어오는 값이 매우 크고, Conquer 과정 내에서 다시 Divide 하는 문제들이 많기 때문에 이를 **재귀(Recursion)**를 활용하여 하나의 함수로 합치는 것입니다. 

> *문제 P를 분할 정복으로 시작했다고 해서, 반드시 서브문제 p1, p2, p3도 분할 정복으로 풀어야하는 것은 아니다! - 위 블로그 발췌*

그렇기에 **분할 정복은 재귀와 같지 않다**는 것을 기억해두시면 좋을 것 같습니다. 분할 정복을 이용하는 가장 기본적인 문제를 한번 보겠습니다.  

+ [2630번: 색종이 만들기](https://www.acmicpc.net/problem/2630)

위 문제에 대한 풀이는 이미 블로그에 있기에 분할 정복을 사용하는 부분만 보도록 하겠습니다.  

```python
# 분할 정복
def divideAndConquer(N, y, x):  # N: 색종이의 한 변 길이, (y, x): 시작 좌표
  # Conquer  
  # 해당 색종이가 모두 같은 색인지 아닌지 확인
  # if ...

  # Divide  
  # 4등분하여 각각의 색종이를 확인, 등분
  divideAndConquer(N//2, y, x)              # 왼쪽 위
  divideAndConquer(N//2, y, x+N//2)         # 오른쪽 위
  divideAndConquer(N//2, y+N//2, x)         # 왼쪽 아래
  divideAndConquer(N//2, y+N//2, x+N//2)    # 오른쪽 아래
```

위의 문제도 Divide와 Conquer과정이 합쳐져 있습니다. 하지만 원래는 아래와 같을 것입니다.  

```python
def divide(N, y, x):
    # 전체 색종이를 4등분
    conquer(N//2, y, x)              # 왼쪽 위
    conquer(N//2, y, x+N//2)         # 오른쪽 위
    conquer(N//2, y+N//2, x)         # 왼쪽 아래
    conquer(N//2, y+N//2, x+N//2)    # 오른쪽 아래

def conquer(N, y, x):
    # 해당 색종이가 모두 같은 색인지 아닌지 확인
    # if ...

    # 모두 같은 색이 아니라면 다시 4등분
    divide(N, y, x)
```

하지만 conquer 과정에서도 divide가 이루어지기 때문에 둘을 합쳐서 더 간결하게 표현한 것입니다. 이와 같이 divide와 conquer를 합쳐서 푸는 문제들이 많지만 원래의 분할 정복은 둘을 분리하여서 생각해야하는 것을 잊지 않으셨으면 좋겠습니다.
